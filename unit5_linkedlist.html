Linked List 鏈結串列

動態記憶體的藝術 — 用指標串連的節點序列

💡 核心定義：Linked List 是由一系列節點 (Node) 組成的線性資料結構，每個節點包含資料和指向下一個節點的指標。節點在記憶體中不需要連續。

每個節點包含兩個部分：

C++
struct Node {
    int data;        // 儲存的資料
    Node* next;      // 指向下一個節點的指標
};

// 建立節點
Node* node = new Node();
node->data = 42;
node->next = nullptr;

💡 關鍵洞察：Linked List 的優勢在於「已知位置」時的插入/刪除只需 O(1)。但「找到位置」本身需要 O(n)。所以實際使用時，要考慮完整操作的時間。

C++ 實作

從零開始建構 Singly Linked List

節點結構

C++
struct Node {
    int data;
    Node* next;
    
    // 建構函式
    Node(int val) : data(val), next(nullptr) {}
};

Linked List 類別

C++
class LinkedList {
private:
    Node* head;
    Node* tail;
    int size;

public:
    LinkedList() : head(nullptr), tail(nullptr), size(0) {}
    
    // 頭部插入 O(1)
    void insertHead(int val) {
        Node* newNode = new Node(val);
        newNode->next = head;
        head = newNode;
        if (!tail) tail = head;  // 空串列時
        size++;
    }
    
    // 尾部插入 O(1)
    void insertTail(int val) {
        Node* newNode = new Node(val);
        if (!tail) {
            head = tail = newNode;
        } else {
            tail->next = newNode;
            tail = newNode;
        }
        size++;
    }
    
    // 刪除頭部 O(1)
    void deleteHead() {
        if (!head) return;
        Node* temp = head;
        head = head->next;
        delete temp;
        if (!head) tail = nullptr;
        size--;
    }
};

常見操作實作

C++
// 搜尋 O(n)
Node* search(int val) {
    Node* curr = head;
    while (curr) {
        if (curr->data == val) return curr;
        curr = curr->next;
    }
    return nullptr;
}

// 在指定節點後插入 O(1)
void insertAfter(Node* prev, int val) {
    if (!prev) return;
    Node* newNode = new Node(val);
    newNode->next = prev->next;
    prev->next = newNode;
    if (prev == tail) tail = newNode;
    size++;
}

// 遍歷列印 O(n)
void print() {
    Node* curr = head;
    while (curr) {
        std::cout << curr->data;
        if (curr->next) std::cout << " -> ";
        curr = curr->next;
    }
    std::cout << " -> NULL" << std::endl;
}
⚠️ 記憶體管理：在 C++ 中使用 new 建立節點後，記得在刪除時用 delete 釋放記憶體。否則會造成 Memory Leak！

使用 STL: std::list

C++
#include <list>

std::list<int> lst;

lst.push_front(10);  // 頭部插入
lst.push_back(20);   // 尾部插入
lst.pop_front();      // 刪除頭部
lst.pop_back();       // 刪除尾部

// 遍歷
for (int x : lst) {
    std::cout << x << " ";
}

// std::list 是 Doubly Linked List！

答對 0 / 5 題
Q1. Linked List 相比 Array 的主要優勢是？
A隨機存取更快
B記憶體使用更少
C在已知位置插入/刪除只需 O(1)
DCache 效能更好
Linked List 在已知位置時，插入/刪除只需修改指標，不需要搬移元素。但隨機存取、記憶體效率、Cache 效能都不如 Array。
Q2. 在 Singly Linked List 中，刪除尾部節點的時間複雜度是？
AO(1)
BO(n)
CO(log n)
DO(n²)
Singly Linked List 只有 next 指標，要刪除尾部必須找到倒數第二個節點來更新它的 next。這需要從頭走訪，所以是 O(n)。
Q3. 以下哪種場景最適合使用 Linked List？
A需要頻繁用索引存取第 i 個元素
B資料量固定且需要最佳 Cache 效能
C需要 Binary Search 快速搜尋
D需要頻繁在清單前端插入/刪除
Linked List 的頭部插入/刪除是 O(1)，非常適合這種場景。Array 在頭部操作需要 O(n) 搬移所有元素。
Q4. Doubly Linked List 相比 Singly Linked List 的優勢是？
A可以 O(1) 刪除尾部節點
B使用更少的記憶體
C隨機存取更快
DCache 效能更好
Doubly Linked List 有 prev 指標，可以直接找到前一個節點，因此刪除尾部只需 O(1)。代價是每個節點多一個指標的空間。
Q5. 下列程式碼執行後，head 指向的節點的 data 值是？
C++
Node* head = new Node(10);
head->next = new Node(20);
Node* newNode = new Node(5);
newNode->next = head;
head = newNode;
A10
B5
C20
D程式會錯誤
這是頭部插入的操作！最後 head = newNode，所以 head 指向 data=5 的節點。完整結構是 5 → 10 → 20 → NULL。
