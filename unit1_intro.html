什麼是資料結構？

學習如何有效地組織和儲存資料，這是所有程式設計的基石

💡 核心定義：資料結構是在電腦中組織、管理和儲存資料的方式，使得資料可以被有效地存取和修改。

為什麼重要？

效能差異

同樣的問題，選擇不同的資料結構可能導致執行時間相差數千倍。

面試必考

90% 以上的技術面試都會涉及資料結構與演算法。

資料 vs 資料結構 vs 演算法

資料 (Data) 原始的值，例如：數字、文字、圖片
資料結構 組織資料的容器，決定資料如何被儲存和存取
演算法 操作資料的步驟，例如排序、搜尋

程式 = 資料結構 + 演算法

靜態 vs 動態

靜態 STATIC

大小在編譯時固定

例如：C/C++ 的 int arr[100]

✓ 存取快、記憶體連續
✗ 無法調整大小

動態 DYNAMIC

大小在執行時可變

例如：C++ 的 vector<int>

✓ 彈性高、自動管理
✗ 可能有額外開銷

Big O 時間複雜度

用數學符號描述演算法的效率，這是評估資料結構的關鍵

📐 Big O 符號描述的是當輸入規模 n 趨近無限大時，演算法執行時間的成長趨勢，忽略常數項和低階項。

常見複雜度比較

O(1)
常數
O(log n)
對數
O(n)
線性
O(n log n)
線性對數
O(n²)
平方
O(2ⁿ)
指數（災難）

實際數字感受

假設 n = 1,000,000（一百萬），且電腦每秒執行 10⁹ 次運算：

複雜度運算次數執行時間感受
O(1)11 ns瞬間
O(log n)2020 ns瞬間
O(n)10⁶1 ms很快
O(n log n)2×10⁷20 ms可接受
O(n²)10¹²17 分鐘慢
O(2ⁿ)無限大宇宙毀滅不可行

程式碼範例

C++
// O(1) — 常數時間
int x = arr[5];  // 直接存取

// O(n) — 線性時間
for (int i = 0; i < n; i++)
    sum += arr[i];  // 遍歷一次

// O(n²) — 平方時間
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        // 巢狀迴圈

// O(log n) — 對數時間
while (left <= right) {
    mid = (left + right) / 2;  // Binary Search
    // 每次排除一半
}
⚡ 記憶技巧：看迴圈！單層 for → O(n)；雙層巢狀 → O(n²)；每次減半 → O(log n)；無迴圈 → O(1)

抽象資料型別 (ADT)

資料結構的藍圖 — 定義「做什麼」，而非「怎麼做」

📋 ADT (Abstract Data Type) 定義了一組資料和可以對它執行的操作，但不指定如何實作。就像汽車的規格表，不管引擎怎麼設計，都要能「加速」和「煞車」。

ADT vs 資料結構

ADT（抽象）

介面 — 定義操作

例：List ADT
• add(item)
• remove(index)
• get(index)
• size()

資料結構（具體）

實作 — 決定如何做

例：實作 List ADT
• ArrayList（陣列）
• LinkedList（鏈結）
• 同樣的操作，不同效能

常見 ADT 及其實作

ADT核心操作可能的實作
List add, remove, get Array, LinkedList
Stack push, pop, peek Array, LinkedList
Queue enqueue, dequeue Array, LinkedList, Circular Buffer
Map / Dictionary put, get, remove Hash Table, Tree Map
Set add, contains, remove Hash Set, Tree Set
Priority Queue insert, extractMin Heap, Sorted Array

為什麼要分開？

🔄

可替換性

換實作不改邏輯

🧪

可測試性

用假實作測試

📚

可理解性

先懂概念再學實作

C++
// 使用 ADT 的思維：只關心操作，不關心實作

// Stack ADT — 無論底層是什麼，都支援這些操作
stack.push(10);      // 放入
stack.push(20);
int top = stack.top();  // 查看頂端 → 20
stack.pop();           // 移除頂端

// C++ STL 已經幫你實作好了！
#include <stack>
std::stack<int> s;  // 直接使用

知識測驗

確認你對資料結構基礎概念的掌握程度

答對 0 / 5 題
Q1. 下列哪個最能描述「資料結構」？
A一種程式語言
B組織和儲存資料的方式
C解決問題的步驟
D電腦的記憶體
資料結構是組織、管理和儲存資料的方式，讓資料可以被有效存取和修改。演算法才是「解決問題的步驟」。
Q2. 在停車場裡，每個車位都有編號，可以直接開到對應位置。這最像哪種資料結構？
AArray（陣列）
BStack（堆疊）
CQueue（佇列）
DTree（樹）
陣列的特性就是可以用索引（編號）直接存取任意位置，就像停車場可以直接開到某個編號的車位。
Q3. O(n²) 和 O(n) 相比，當 n = 1000 時：
AO(n²) 比較快
B差不多快
CO(n²) 慢 1000 倍
D無法比較
O(n) = 1000 次運算，O(n²) = 1000×1000 = 1,000,000 次運算，相差 1000 倍。這就是為什麼選對資料結構很重要！
Q4. 以下哪個是「非線性」資料結構？
AArray
BStack
CQueue
DTree
Tree（樹）是非線性結構，因為一個節點可以有多個子節點（一對多關係）。Array、Stack、Queue 都是線性結構。
Q5. ADT (Abstract Data Type) 定義的是：
A具體的記憶體配置方式
B資料可執行的操作（介面）
C程式的執行效率
D變數的命名規則
ADT 定義的是「做什麼」（操作/介面），而不是「怎麼做」（實作）。同一個 ADT 可以有多種不同的資料結構來實作。
