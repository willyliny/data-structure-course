實際應用

Stack 在程式設計中無處不在

1. 函式呼叫堆疊 (Call Stack)

📞 每當呼叫函式，系統會將函式資訊 push 到 Call Stack；函式返回時 pop。這就是為什麼遞迴太深會造成 Stack Overflow！
C++
void c() { // 第三層 }

void b() { 
    c();  // 呼叫 c
}

void a() { 
    b();  // 呼叫 b
}

int main() {
    a();  // 呼叫 a
}

Call Stack 狀態

main() line 11
a() line 7
b() line 4
c() line 1

C++ 實作

使用 Array 實作 Stack — 簡單又高效

用 Array 實作

C++
class Stack {
private:
    int arr[1000];     // 固定大小的陣列
    int topIndex;       // 頂端位置

public:
    Stack() : topIndex(-1) {}   // 初始為空
    
    // Push: O(1)
    void push(int x) {
        if (topIndex >= 999) {
            throw runtime_error("Stack Overflow!");
        }
        arr[++topIndex] = x;
    }
    
    // Pop: O(1)
    int pop() {
        if (isEmpty()) {
            throw runtime_error("Stack Underflow!");
        }
        return arr[topIndex--];
    }
    
    // Top/Peek: O(1)
    int top() {
        if (isEmpty()) {
            throw runtime_error("Stack is empty!");
        }
        return arr[topIndex];
    }
    
    bool isEmpty() { return topIndex == -1; }
    int size() { return topIndex + 1; }
};
⚠️ 限制：Array 實作的 Stack 有固定大小。如果需要動態大小，可以用 vector 或 Linked List 實作。

使用 STL: std::stack

C++
#include <stack>
using namespace std;

stack<int> s;

s.push(10);      // 放入 10
s.push(20);      // 放入 20
s.push(30);      // 放入 30

s.top();        // 回傳 30（不移除）
s.pop();        // 移除頂端（無回傳值！）
s.top();        // 現在是 20

s.empty();      // false
s.size();       // 2
⚠️ 注意！STL 的 pop() 不會回傳值！要先用 top() 取值，再 pop() 移除。

實際使用範例

C++
// 括號匹配
bool isValid(string s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(' || c == '[' || c == '{') {
            st.push(c);
        } else {
            if (st.empty()) return false;
            char top = st.top(); st.pop();
            if (c == ')' && top != '(') return false;
            if (c == ']' && top != '[') return false;
            if (c == '}' && top != '{') return false;
        }
    }
    return st.empty();
}

時間複雜度

Stack 的所有操作都是常數時間

操作Array 實作Linked List 實作
push(x)O(1)O(1)
pop()O(1)O(1)
top()O(1)O(1)
isEmpty()O(1)O(1)
size()O(1)O(1)
✨ 完美的 O(1)！Stack 是少數所有操作都是 O(1) 的資料結構。這是因為我們只在一端（頂端）操作。

空間複雜度

實作方式空間複雜度特點
Array（固定大小）O(n)預先分配，可能浪費空間
vector（動態）O(n)均攤 O(1) 的動態擴容
Linked ListO(n)每個元素額外需要指標空間

Array vs Linked List 實作比較

ARRAY 實作

✓ Cache 友好（連續記憶體）

✓ 無額外指標開銷

✗ 大小固定（除非用 vector）

✗ 可能浪費空間

LINKED LIST 實作

✓ 動態大小，用多少分配多少

✓ 永不 overflow（除非記憶體耗盡）

✗ 每節點額外指標開銷

✗ Cache 不友好

💡 實務建議：大多數情況使用 std::stack（底層是 deque）即可。如果你確定最大容量，Array 實作更簡單高效。

知識測驗

確認你對 Stack 的掌握程度

答對 0 / 5 題
Q1. Stack 的特性是？
A先進先出 (FIFO)
B後進先出 (LIFO)
C隨機存取
D優先順序處理
Stack 是 LIFO (Last In, First Out)，最後放入的元素最先被取出。就像疊書，最後放上去的書最先被拿走。
Q2. 執行以下操作後，stack.top() 的值是？
push(5) → push(10) → push(15) → pop() → push(20)
A5
B10
C15
D20
push(5) push(10) push(15) 後 Stack 是 [5,10,15]（15 在頂）。pop() 移除 15，變成 [5,10]。push(20) 後變成 [5,10,20]，top 是 20。
Q3. 以下哪個不是 Stack 的常見應用？
A函式呼叫堆疊 (Call Stack)
B括號匹配
C列印佇列（先送先印）
D撤銷操作 (Undo)
列印佇列是「先送先印」，這是 FIFO 特性，應該用 Queue 而不是 Stack。Stack 適合需要「後進先出」的場景。
Q4. 在用 Array 實作的 Stack 中，push 操作的時間複雜度是？
AO(1)
BO(n)
CO(log n)
DO(n²)
push 只需要將 topIndex 加 1，然後在該位置寫入值。這是常數時間操作，所以是 O(1)。
Q5. C++ STL 的 std::stack，呼叫 pop() 會：
A回傳並移除頂端元素
B只移除頂端元素，不回傳
C只回傳頂端元素，不移除
D移除所有元素
STL 的 pop() 只移除頂端元素，不回傳值！如果需要值，要先呼叫 top() 取得，再呼叫 pop() 移除。這是常見的陷阱。
